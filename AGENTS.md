# earnwithhermes.com

Astro 6 + Tailwind 4 + Svelte 5 site. Live at https://earnwithhermes.com (Nginx, SSL).
GitHub: https://github.com/taku-ops/earnwithhermes

## Quick start

```bash
npm run dev        # Dev server on localhost:4321
npm run build      # Build to dist/
npm run preview    # Preview production build
```

## Deploy

```bash
npm run build
sudo rsync -a --delete dist/ /var/www/html/earnwithhermes.com/
sudo chown -R www-data:www-data /var/www/html/earnwithhermes.com/
```

## Git

Commit and push after every change to the live site. The repo is the source of truth.

```bash
git add -A
git commit -m "description of changes"
git push
```

## Content conventions

### Article writing strategy

Use Hermes Agent docs as research, but write original content from personal experience. Never paraphrase the docs.

- **Docs answer:** "what is this feature?"
- **Our articles answer:** "what happened when I used this feature to make money?"

These are different search intents. Google rewards the latter.

### Article types

| Type | Description |
|------|-------------|
| Tutorial | Step-by-step, teach a specific skill |
| Blueprint | Pattern / strategy, show an architecture |
| Opinion | Analysis / hot take (not doing yet) |

### SEO differentiation

- Beginner-friendly tone is table stakes
- Unique differentiator: personal experience, real numbers, specific setups, results
- If it reads like a doc page rewritten for beginners, it's not original enough

## Stats bar (homepage)

Every number displayed in the stats bar must trace back to a specific story on the site. When a story is drafted, check if any stat became orphaned.
